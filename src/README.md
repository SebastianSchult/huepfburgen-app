# Source Code

Application source code lives here.

## Structure

- `app/pages`: route-level pages
- `app/features`: reusable feature components and dialogs
- `app/services`: Firebase and domain services
- `app/guards`: route protection
- `app/models`: shared TypeScript data shapes
- `assets`: static assets and fonts

## Rules

- Keep Angular components focused.
- Keep Firestore access in services.
- Keep models aligned with Firestore rules.
- Preserve existing standalone component patterns.
- Update root documentation when behavior, architecture, deployment, QA, or security expectations change.
